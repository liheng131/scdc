# /push 命令

## 描述
将代码上传到代码仓库，支持自定义提交描述。

## 使用方法
- `/push` - 使用自动生成的提交描述
- `/push 你的提交描述` - 使用自定义提交描述

## 执行步骤

### 1. 检查远程仓库绑定状态
```bash
git remote -v
```

### 2. 判断远程仓库
**如果没有任何远程仓库输出**：
- 停止执行
- 提示用户：`未检测到远程仓库绑定。请提供远程仓库地址（如：https://github.com/yourname/repo.git 或 git@github.com:yourname/repo.git）`
- 等待用户输入后执行：`git remote add origin <用户提供的地址>`

**如果有远程仓库**：
- 继续使用已绑定的 `origin` 远程仓库

### 3. 获取当前分支
```bash
git branch --show-current
```

### 4. 暂存所有更改
```bash
git add -A
```

### 5. 生成交替描述
- **如果用户提供了描述**（如 `/push 修复登录问题`）：使用用户提供的描述
- **如果用户未提供描述**：执行以下步骤生成描述
  1. 运行 `git status --short` 获取变更文件列表
  2. 运行 `git diff --staged --stat` 获取变更统计
  3. 运行 `git diff --staged` 查看具体变更内容
  4. 根据变更内容自动生成简洁的提交描述（1-2 句话，描述做了什么）

### 6. 提交代码
```bash
git commit -m "用户提供的描述或自动生成的描述"
```

### 7. 推送代码
```bash
git push origin <当前分支名>
```

### 8. 反馈结果
- 成功：显示推送结果和提交描述
- 失败：显示错误信息并建议用户检查
